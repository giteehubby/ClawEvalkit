A — Task Understanding
Agent misunderstands task goals or drifts from objective during execution
B — Tool Grounding
Correct intent but wrong tool, parameters, or output format for execution context
C — Memory / State
Fails to maintain context, intermediate facts, or prior search results across steps
D — Verification
No self-checking or error recovery; proceeds past errors without correction
E — Procedural Skill
Missing domain-specific procedural knowledge or step-level skill; knows correct plan but cannot execute it correctly

🔍 Detailed Failure Case Studies
Type E task_13_image_gen — AI Image Generation 93.7% Failure Rate (15/16 runs)
Profile: Score 0.00–0.50 across all runs. Status = timeout. Agent repeatedly attempts to access non-existent image generation tools.
"The agent completely failed to generate the requested image. They made multiple unsuccessful attempts to access image generation tools/APIs and created irrelevant files instead."
"No generate_image tool exists in the available toolset. Agent attempted a reasonable prompt but the image tool is for analysis, not generation."
Diagnosis: The nanobot runtime does not have an image generation tool in its default tool set. The agent attempts to use non-existent tools, times out, and receives no partial credit.
Type E Infrastructure Fix Required Prompting: No Effect
Type A task_15_daily_summary — Daily Research Briefing 75% Failure Rate (12/16 runs)
Profile: Score 0.00 across all runs and all model checkpoints. Agent creates AGENTS.md / HEARTBEAT.md instead of reading the research/ directory.
"Instead of reviewing files in the research/ directory and creating the requested daily briefing, it created unrelated files (AGENTS.md, HEARTBEAT.md) and performed none of the steps necessary to produce the executive daily briefing."
"The agent correctly discovered and read all 5 research files...demonstrating proper file exploration. However, the transcript shows no write action to create daily_briefing.md — the core deliverable."
Diagnosis: The nanobot initialization process creates AGENTS.md and HEARTBEAT.md in the workspace before task execution begins. The agent interprets these files as task context and spends its tool call budget maintaining these files instead of reading research/.
Type A Infrastructure-Behavior Interaction Isolate Init from Task Execution
Type B task_10_workflow — Multi-step API Workflow 37% Failure Rate (6/16 runs)
Profile: Score 0.42 on exp87_ratio5pct_step660. Agent generates syntactically correct Python script but does not read config.json for API credentials.
"Automated checks: read_config=0.0, script_created=1.0, valid_syntax=1.0. LLM Judge: script_quality=0.0, documentation_quality=0.0, process_understanding=0.0."
Diagnosis: B-type failure — the agent knows it should generate a Python script for the workflow, but does not know it must first read config.json for API credentials and endpoint URLs. The generated code is syntactically valid but semantically useless.
Type B Plan-First Prompting: Potentially Effective Preflight Check Recipe: Applicable
