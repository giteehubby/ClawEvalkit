Tribe 迁移示例

# === 迁移前 (call_llm 裸 API) === from clawevalkit.utils.api import call_llm response = call_llm( [{"role": "user", "content": test["prompt"]}], config, max_tokens=2048 ) # === 迁移后 (NanoBotAgent) === from ..utils.nanobot import import_nanobot_agent NanoBotAgent = import_nanobot_agent() agent = NanoBotAgent( model=config["model"], api_url=config["api_url"], api_key=config["api_key"], workspace=workspace, timeout=60 ) result = agent.execute(test["prompt"], session_id=f"tribe_{model_key}_{test['id']}") response = result.content or ""

SkillsBench 迁移 — 最大变化

迁移前需要 4 个辅助函数手动处理代码提取和执行，迁移后全部由 NanoBotAgent 内置 tools 替代：

# 删除的函数 (由 NanoBotAgent 内置 tools 替代) def _call_llm_multi() # → NanoBotAgent.execute() def _extract_files() # → NanoBotAgent 用 WriteFile 直接写 def _write_files() # → NanoBotAgent 用 WriteFile 直接写 def _execute_scripts() # → NanoBotAgent 用 Exec 直接跑 # 保留的函数 (harness 仍需外部验证) def _setup_workspace() # 复制环境文件 def _run_pytest() # 外部 pytest 验证 def _list_workspace_files() # 反馈循环

PinchBench 迁移 — 完全重写

从依赖外部不可用模块 (lib_agent, lib_grading) 改为自包含：

# 新增: _load_tasks() — 解析 task markdown def _load_tasks(self): # 1. 解析 YAML frontmatter (--- ... ---) # 2. 提取 ## Prompt 部分 → agent 指令 # 3. 提取 ## Automated Checks 中的 ```python grade()``` 函数 # 新增: _run_grade() — exec 执行内嵌评分 def _run_grade(self, grade_code, transcript, workspace_path): namespace = {} exec(grade_code, namespace) return namespace["grade"](transcript, workspace_path)

修复: ExecTool TypeError

NanoBotAgent 初始化 ExecTool 时传了不支持的 workspace 参数：

# OpenClawPro/harness/agent/nanobot.py:190 # Before: ExecTool(timeout=self.timeout, working_dir=str(self.workspace), workspace=self.workspace) # After: ExecTool(timeout=self.timeout, working_dir=str(self.workspace))
